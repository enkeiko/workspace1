# Phase 1 상세 기획서: 광고대행사 관리 시스템 MVP

**프로젝트명**: 42ment 광고대행사 관리 시스템
**Phase**: 1 (MVP)
**버전**: 1.0
**작성일**: 2025-11-14
**개발 기간**: 3-4개월

---

## 📌 Phase 1 개요

### 목표
1인 또는 소규모 광고대행사가 일상적인 업무(고객 관리, 매장 관리, 판매, 정산)를 하나의 시스템에서 관리할 수 있는 **최소 기능 제품(MVP)**을 구축합니다.

### 핵심 가치 제안
1. **통합 관리**: 고객-매장-주문-정산을 하나의 시스템에서 관리
2. **시간 절약**: 견적서/세금계산서 생성 시간 80% 단축
3. **재무 투명성**: 미수금 현황을 실시간으로 파악
4. **고객 서비스 향상**: 광고 계정 정보 및 보고서 발송을 체계적으로 관리

### Phase 1 범위
- ✅ 내정보, 고객, 매장, 상품 관리
- ✅ 견적, 주문, 계약서 관리
- ✅ 기본 정산 (세금계산서, 입금, 미수금)
- ✅ 광고 계정 정보 관리 (고객 연동)
- ✅ 보고서 관리 (고객 연동)
- ✅ 간단한 대시보드

### Phase 1 제외 항목
- ❌ OCR 자동화 (수동 입력으로 대체)
- ❌ 크롤러 연동 (수동 데이터 입력)
- ❌ 광고 API 연동 (수동 데이터 입력)
- ❌ 은행 API 연동 (수동 입금 체크)
- ❌ 자동 알림/발송 (수동 발송)
- ❌ 프로젝트/캘린더 통합
- ❌ 시간 추적 기능

## 🎯 기능 요구사항 (Functional Requirements)

### 1. 내정보 관리

**FR-001**: 회사 기본 정보 관리
- 상호, 대표자명, 사업자번호
- 주소, 전화번호, 이메일
- 사업자등록증 파일 업로드

**FR-002**: 회사 계좌 정보 관리
- 은행명, 계좌번호, 예금주
- 여러 계좌 등록 가능
- 기본 계좌 설정

---

### 2. 고객 관리

**FR-010**: 고객 기본 정보 CRUD
- 고객 등록: 회사명, 대표자, 담당자명, 연락처, 이메일, 사업자번호, 주소, 메모
- 고객 목록 조회: 전체 목록, 페이지네이션
- 고객 검색: 회사명, 담당자명, 연락처로 검색
- 고객 상세 정보 조회
- 고객 정보 수정
- 고객 삭제 (연관 데이터 있으면 삭제 불가, 경고 표시)

**FR-011**: 광고 계정 정보 관리 (고객별)
- 매체별 계정 정보 등록: 네이버, 구글, 메타, 카카오 등
- 계정 ID, 비밀번호 (암호화 저장), API 키 (암호화 저장)
- 접근 권한 정보 (읽기/쓰기)
- 계정 만료일 설정
- 계정 상태 (활성/비활성)
- 계정별 메모

**FR-012**: 보고서 발송 정보 관리 (고객별)
- 보고서 발송 주기: 주간/월간/분기별
- 발송 요일 또는 일자 (예: 매주 월요일, 매월 5일)
- 수신 이메일 목록 (쉼표로 구분)
- 보고서 템플릿 선택 (주간 리포트/월간 리포트)
- 발송 여부 (활성/비활성)

**FR-013**: 고객 목록 다운로드
- CSV, Excel 형식으로 다운로드

---

### 3. 매장 관리

**FR-020**: 매장 기본 정보 CRUD
- 매장 등록: 매장명, 주소, 업종, 전화번호, 고객 연결, 메모
- 매장 목록 조회: 전체 목록, 고객별 필터
- 매장 검색: 매장명, 주소로 검색
- 매장 상세 정보 조회
- 매장 정보 수정
- 매장 삭제 (연관 데이터 있으면 삭제 불가)

**FR-021**: 매장 목록 다운로드
- CSV, Excel 형식으로 다운로드

---

### 4. 판매 관리 - 상품

**FR-030**: 상품 카테고리 관리
- 카테고리 등록: 블로그, 카페, 지도 최적화, 키워드 광고 등
- 카테고리별 기본 단가
- 카테고리 설명 및 메모

**FR-031**: 상품 관리 (선택사항)
- 상품 등록: 상품명, 카테고리, 기본 단가, 설명
- 상품 목록 조회
- 상품 수정/삭제

---

### 5. 판매 관리 - 견적/주문

**FR-040**: 견적서 생성
- 고객 선택 (필수)
- 매장 선택 (선택사항)
- 상품 선택 및 수량
- 단가 자동 불러오기 (고객별 단가 우선, 없으면 기본 단가)
- 단가 수동 수정 가능
- 고객별 단가 저장 옵션
- 공급가, 부가세, 합계 자동 계산
- 부가세 포함/별도 옵션
- 견적 메모
- 견적서 PDF 다운로드

**FR-041**: 견적서 목록 및 관리
- 견적서 목록 조회 (날짜순, 고객별, 상태별)
- 견적서 상태: 대기/승인/거절/주문전환
- 견적서 상세 조회
- 견적서 수정 (주문 전환 전까지만)
- 견적서 삭제

**FR-042**: 주문 등록
- 견적서에서 주문 전환
- 또는 직접 주문 등록 (견적서와 동일한 입력 방식)
- 주문 상태: 대기/진행/완료/취소
- 주문일, 납품 예정일

**FR-043**: 주문 목록 및 조회
- 주문 목록 조회 (날짜순, 고객별, 매장별, 상품별, 상태별)
- 일/주/월/분기/연 단위 조회
- 주문 상세 정보
- 주문 상태 변경
- 주문 수정 (완료 전까지만)
- 주문 취소

**FR-044**: 주문 목록 다운로드
- CSV, Excel 형식으로 다운로드
- 필터 조건 반영

---

### 6. 판매 관리 - 계약서

**FR-050**: 계약서 관리
- 계약서 파일 업로드 (PDF, 이미지)
- 계약 정보 등록
  - 고객 연결 (필수)
  - 매장 연결 (선택)
  - 계약명
  - 계약 기간 (시작일/종료일)
  - 계약 금액
  - 계약 내용 요약 (textarea)
- 계약서 목록 조회
- 계약서 상세 조회
- 계약서 수정/삭제

**FR-051**: 계약 만료 알림
- 계약 만료 D-30, D-7인 계약서 목록 표시
- 대시보드에 알림 배지 표시

---

### 7. 정산 관리

**FR-060**: 세금계산서 발행 기록
- 세금계산서 정보 등록
  - 주문 연결 (선택사항)
  - 고객 연결 (필수)
  - 발행일
  - 공급가, 부가세, 합계
  - 세금계산서 번호
  - 파일 첨부 (선택)
- 세금계산서 목록 조회
- 세금계산서 상세 조회
- 세금계산서 수정/삭제

**FR-061**: 입금 관리
- 입금 정보 등록
  - 세금계산서 연결 (선택)
  - 고객 연결 (필수)
  - 입금일
  - 입금 금액
  - 입금 계좌
  - 메모
- 입금 목록 조회
- 입금 상세 조회
- 입금 수정/삭제
- 세금계산서와 입금 매칭
  - 세금계산서에서 입금 체크 표시
  - 미수금 자동 계산

**FR-062**: 미수금 현황
- 고객별 미수금 조회
  - 발행 세금계산서 합계
  - 입금 합계
  - 미수금 (발행액 - 입금액)
- 전체 미수금 요약
- 기간별 미수금 조회
- 미수금 목록 다운로드

**FR-063**: 매출 요약
- 월별 매출 요약
  - 총 매출 (세금계산서 발행액)
  - 입금액
  - 미수금
- 고객별 매출 조회
- 상품별 매출 조회
- 기간별 매출 조회 (일/주/월/분기/연)
- 매출 데이터 다운로드

### 8. 보고서 관리

**FR-070**: 보고서 생성
- 고객 선택 (필수)
- 매장 선택 (선택, 1개 고객에 여러 매장인 경우)
- 보고 기간 선택 (시작일 ~ 종료일)
- 템플릿 선택 (주간/월간)
- 보고서 데이터 수동 입력
  - 성과 데이터 (노출, 클릭, 전환, 비용 등)
  - 키워드 순위 변화
  - 리뷰 수/평점 변화
  - 주요 성과 요약 (textarea)
  - 개선 사항 및 특이사항 (textarea)
- 파일 첨부 (이미지, PDF 등)
- 보고서 저장

**FR-071**: 보고서 목록 및 관리
- 보고서 목록 조회 (날짜순, 고객별, 매장별)
- 보고서 상세 조회
- 보고서 수정
- 보고서 삭제
- 보고서 PDF 다운로드

**FR-072**: 보고서 발송 이력
- 발송 일시 기록
- 발송 대상 (이메일 목록)
- 발송 방식 (이메일/카카오톡/수동)
- 발송 상태 (성공/실패)

**FR-073**: 보고서 목록 다운로드
- CSV, Excel 형식

### 10. 공통 기능

- 단일 사용자 비밀번호 인증
- 로그인/로그아웃
- 비밀번호 변경

**FR-091**: 데이터 다운로드
- 모든 목록 조회 화면에서 CSV/Excel 다운로드 지원
- 필터/검색 조건 반영

## 🗂️ 핵심 엔티티 (Key Entities)

### 1. CompanyInfo (내정보)
회사의 기본 정보를 저장합니다.

**속성**:
- `id` (PK)
- `company_name`: 상호
- `ceo_name`: 대표자명
- `business_number`: 사업자번호
- `address`: 주소
- `phone`: 전화번호
- `email`: 이메일
- `business_registration_file`: 사업자등록증 파일 경로
- `created_at`, `updated_at`, `created_by`

---

### 2. BankAccount (회사 계좌)
회사의 은행 계좌 정보를 저장합니다.

**속성**:
- `id` (PK)
- `bank_name`: 은행명
- `account_number`: 계좌번호
- `account_holder`: 예금주
- `is_default`: 기본 계좌 여부 (Y/N)
- `created_at`, `updated_at`, `created_by`

---

### 3. Client (고객)
광고대행사의 고객(클라이언트) 정보를 저장합니다.

**속성**:
- `id` (PK)
- `company_name`: 회사명
- `ceo_name`: 대표자
- `contact_name`: 담당자명
- `phone`: 연락처
- `email`: 이메일
- `business_number`: 사업자번호
- `address`: 주소
- `memo`: 메모
- `report_frequency`: 보고서 발송 주기 (weekly/monthly/quarterly/none)
- `report_day`: 발송 요일 또는 일자 (예: "Monday", "5")
- `report_emails`: 수신 이메일 목록 (JSON 배열 또는 쉼표 구분)
- `report_template`: 보고서 템플릿 (weekly_report/monthly_report)
- `report_enabled`: 보고서 발송 활성화 (Y/N)
- `created_at`, `updated_at`, `created_by`

**관계**:
- 1:N → Store (고객 1명에 여러 매장)
- 1:N → AdAccount (고객 1명에 여러 광고 계정)
- 1:N → Order (고객 1명에 여러 주문)
- 1:N → Contract (고객 1명에 여러 계약)
- 1:N → Report (고객 1명에 여러 보고서)

---

### 4. Store (매장)
고객의 매장 정보를 저장합니다.

**속성**:
- `id` (PK)
- `client_id` (FK → Client)
- `store_name`: 매장명
- `address`: 주소
- `business_type`: 업종
- `phone`: 전화번호
- `memo`: 메모
- `created_at`, `updated_at`, `created_by`

**관계**:
- N:1 → Client
- 1:N → Order (매장 1개에 여러 주문)
- 1:N → Report (매장 1개에 여러 보고서)

---

### 5. AdAccount (광고 계정)
고객별 광고 매체 계정 정보를 저장합니다.

**속성**:
- `id` (PK)
- `client_id` (FK → Client)
- `platform`: 광고 매체 (naver/google/meta/kakao 등)
- `account_id`: 계정 ID
- `password`: 비밀번호 (암호화 저장)
- `api_key`: API 키 (암호화 저장)
- `access_level`: 접근 권한 (read/write/admin)
- `expiry_date`: 계정 만료일
- `status`: 계정 상태 (active/inactive)
- `memo`: 메모
- `created_at`, `updated_at`, `created_by`

**관계**:
- N:1 → Client

---

### 6. ProductCategory (상품 카테고리)
판매 상품의 카테고리를 저장합니다.

**속성**:
- `id` (PK)
- `category_name`: 카테고리명 (예: 블로그, 카페, 지도 최적화)
- `default_price`: 기본 단가
- `description`: 설명
- `created_at`, `updated_at`, `created_by`

**관계**:
- 1:N → Product (선택사항)

---

### 7. Product (상품) - 선택사항
판매 상품을 저장합니다. (Phase 1에서는 선택사항)

**속성**:
- `id` (PK)
- `category_id` (FK → ProductCategory)
- `product_name`: 상품명
- `default_price`: 기본 단가
- `description`: 설명
- `created_at`, `updated_at`, `created_by`

---

### 8. Quote (견적서)
고객에게 발행하는 견적서를 저장합니다.

**속성**:
- `id` (PK)
- `client_id` (FK → Client)
- `store_id` (FK → Store, nullable)
- `quote_number`: 견적서 번호 (자동 생성)
- `quote_date`: 견적일
- `status`: 상태 (pending/approved/rejected/converted)
- `subtotal`: 공급가
- `vat`: 부가세

## 👤 사용자 스토리 (User Stories)

### Story 1: 고객 등록 및 광고 계정 관리
**우선순위**: P1 (최우선)

**As a** 광고대행사 운영자
**I want to** 고객 정보와 함께 광고 계정 정보를 등록하고 관리하고
**So that** 고객별로 어떤 광고 매체를 사용하는지 한눈에 파악할 수 있다

**Acceptance Criteria**:
1. **Given** 로그인한 상태에서, **When** 고객 추가 화면에서 고객 기본 정보를 입력하고 저장하면, **Then** 고객 목록에 새 고객이 표시된다
2. **Given** 고객이 등록된 상태에서, **When** 고객 상세 화면에서 "광고 계정 추가" 버튼을 클릭하고 네이버 광고 계정 정보를 입력하면, **Then** 해당 고객의 광고 계정 목록에 네이버 계정이 표시된다
3. **Given** 광고 계정이 등록된 상태에서, **When** 계정 만료일이 7일 이내로 남았으면, **Then** 대시보드에 알림이 표시된다

---

### Story 2: 견적서 생성 및 주문 전환
**우선순위**: P1 (최우선)

**As a** 광고대행사 운영자
**I want to** 고객에게 견적서를 빠르게 생성하고 주문으로 전환하고
**So that** 수작업으로 견적서를 만드는 시간을 절약할 수 있다

**Acceptance Criteria**:
1. **Given** 고객과 상품이 등록된 상태에서, **When** 견적서 생성 화면에서 고객, 상품, 수량을 선택하면, **Then** 합계가 자동 계산되고 견적서가 생성된다
2. **Given** 견적서가 생성된 상태에서, **When** "PDF 다운로드" 버튼을 클릭하면, **Then** 고객 정보와 상품 목록이 포함된 PDF가 다운로드된다
3. **Given** 견적서 상태가 "승인"인 상태에서, **When** "주문 전환" 버튼을 클릭하면, **Then** 동일한 내용의 주문이 생성되고 견적서 상태가 "주문 전환"으로 변경된다

---

### Story 3: 세금계산서 발행 및 입금 관리
**우선순위**: P1 (최우선)

**As a** 광고대행사 운영자
**I want to** 세금계산서 발행과 입금을 체계적으로 관리하고
**So that** 미수금 현황을 실시간으로 파악할 수 있다

**Acceptance Criteria**:
1. **Given** 주문이 완료된 상태에서, **When** 세금계산서 발행 화면에서 주문을 선택하고 발행일을 입력하면, **Then** 세금계산서가 생성되고 주문 금액이 자동으로 입력된다
2. **Given** 세금계산서가 발행된 상태에서, **When** 입금 등록 화면에서 세금계산서를 선택하고 입금 금액을 입력하면, **Then** 해당 세금계산서의 입금 여부가 "Y"로 변경되고 미수금이 감소한다
3. **Given** 여러 세금계산서와 입금이 등록된 상태에서, **When** 미수금 현황 화면을 열면, **Then** 고객별 미수금이 자동 계산되어 표시된다

---

### Story 4: 보고서 작성 및 발송 이력 관리
**우선순위**: P2 (우선)

**As a** 광고대행사 운영자
**I want to** 고객에게 발송할 보고서를 작성하고 발송 이력을 관리하고
**So that** 어떤 고객에게 언제 보고서를 발송했는지 추적할 수 있다

**Acceptance Criteria**:
1. **Given** 고객과 매장이 등록된 상태에서, **When** 보고서 생성 화면에서 고객, 매장, 보고 기간을 선택하고 성과 데이터를 입력하면, **Then** 보고서가 생성된다
2. **Given** 보고서가 생성된 상태에서, **When** "PDF 다운로드" 버튼을 클릭하면, **Then** 보고서 내용이 포함된 PDF가 다운로드된다
3. **Given** 보고서 상세 화면에서, **When** 발송 정보 (발송 일시, 발송 대상, 발송 방식)를 입력하고 저장하면, **Then** 보고서 발송 이력이 기록된다
4. **Given** 고객 상세 화면에서, **When** "보고서 발송 정보" 섹션을 확인하면, **Then** 해당 고객의 보고서 발송 주기, 발송 일자, 수신 이메일이 표시된다

---

### Story 5: 대시보드에서 핵심 지표 확인
**우선순위**: P3 (선택)

**As a** 광고대행사 운영자
**I want to** 대시보드에서 핵심 지표를 한눈에 확인하고
**So that** 매일 아침 업무를 시작할 때 중요한 정보를 빠르게 파악할 수 있다

**Acceptance Criteria**:
1. **Given** 로그인한 상태에서, **When** 대시보드를 열면, **Then** 이번 달 매출, 입금액, 미수금이 표시된다
2. **Given** 계약 만료가 7일 이내로 남은 계약이 있는 상태에서, **When** 대시보드를 열면, **Then** "계약 만료 알림" 섹션에 해당 계약 목록이 표시된다
3. **Given** 대시보드에서, **When** "최근 주문" 섹션을 확인하면, **Then** 최근 5건의 주문이 날짜순으로 표시된다

## ✅ 성공 기준 (Success Criteria)

### 정량적 성공 기준

**SC-001**: 고객 등록 시간 단축
- 고객 기본 정보 + 광고 계정 1개 등록을 **2분 이내**에 완료

**SC-002**: 견적서 생성 시간 단축
- 상품 3개 포함 견적서 생성 및 PDF 다운로드를 **1분 이내**에 완료
- 기존 수작업 대비 **80% 시간 절약**

**SC-003**: 미수금 파악 시간 단축
- 전체 미수금 현황을 **5초 이내**에 조회

**SC-004**: 보고서 작성 시간 단축
- 보고서 1개 작성 및 PDF 다운로드를 **5분 이내**에 완료

**SC-005**: 데이터 안정성
- 50개 고객, 100개 매장, 200개 주문을 **지연 없이** 관리

**SC-010**: 사용 편의성

**SC-011**: 데이터 일관성
- 고객-매장-주문-정산 데이터가 상호 연동되어 중복 입력 없음

**SC-012**: 업무 효율 향상
- 일일 업무 시간 중 **최소 2시간 절약** (견적서, 정산 관리 자동화)

## 🔒 가정 사항 (Assumptions)

**A-001**: 단일 사용자
- 초기 버전은 단일 사용자(대행사 운영자 본인)만 사용

- 데이터는 PostgreSQL 또는 MySQL에 저장
- 로컬 환경 또는 클라우드(AWS, GCP, Naver Cloud) 배포

**A-003**: 수동 입력 우선
- Phase 1에서는 OCR, API 연동 없음
- 모든 데이터는 수동 입력 또는 엑셀 업로드

**A-004**: PDF 생성
- 견적서, 보고서 PDF는 기본 템플릿으로 생성
- 고급 디자인 커스터마이징은 Phase 2-3에서 지원

**A-005**: 파일 저장
- 파일은 서버 파일 시스템 또는 클라우드 스토리지에 저장
- 파일 크기 제한: 5MB

**A-006**: 암호화
- 광고 계정 비밀번호 및 API 키는 AES-256 또는 bcrypt로 암호화 저장

**A-007**: 부가세 계산
- 부가세는 공급가의 10%로 자동 계산
- 부가세 포함/별도 옵션 제공

**A-008**: 데이터 삭제 정책
- 연관 데이터가 있는 경우 삭제 불가 (Soft Delete 또는 제약 조건)
