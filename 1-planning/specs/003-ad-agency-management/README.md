## 📚 문서 구성

이 폴더는 광고대행사 관리 시스템의 기획 문서를 포함합니다.

### 1. [roadmap.md](./roadmap.md) - 단계별 구축 로드맵
전체 프로젝트를 Phase 1-3로 나누어 단계별 개발 계획을 제시합니다.

**주요 내용**:
- Phase 1 (3-4개월): MVP - 핵심 비즈니스 운영
- Phase 2 (2-3개월): 효율화 & 자동화
- Phase 3 (2-3개월): 고급 기능 & 외부 연동
- 각 Phase별 개발 항목, 난이도, 중요도
- Phase 1 제외 항목 및 이유

---

### 2. [phase1-spec.md](./phase1-spec.md) - Phase 1 상세 기획서
Phase 1 (MVP)의 상세 기능 명세를 포함합니다.

**주요 내용**:
- 기능 요구사항 (Functional Requirements)
  - 내정보 관리
  - 고객 관리 (광고 계정 정보, 보고서 발송 정보 포함)
  - 매장 관리
  - 판매 관리 (상품, 견적, 주문, 계약서)
  - 정산 관리 (세금계산서, 입금, 미수금)
  - 보고서 관리
  - 대시보드
- 핵심 엔티티 (15개 테이블)
- 사용자 스토리 (5개)
- 성공 기준, 가정 사항, 제약 사항
- 개발 우선순위 (Sprint 1-10)

---

### 3. [data-model.md](./data-model.md) - 데이터베이스 스키마 설계
Phase 1의 데이터베이스 스키마를 상세히 설명합니다.

**주요 내용**:
- ERD 개요
- 테이블 정의 (15개)
  - company_info, bank_account
  - client, store, ad_account
  - product_category, product
  - quote, quote_item
  - order, order_item
  - contract, invoice, payment
  - report
- 인덱스 전략
- 보안 및 제약 조건
- 주요 쿼리 예시
- 마이그레이션 전략

## 🎯 핵심 특징

### 원래 아이디어 노트 기능 (모두 포함)
✅ 내정보 관리
✅ 고객 관리 (사업자번호, OCR은 Phase 2)
✅ 매장 관리 (매장 기초정보, 분석은 Phase 3)
✅ 구매 관리 (Phase 2로 이연)
✅ 판매 관리 (상품, 주문, 견적, 계약서)
✅ 매출/정산 관리 (세금계산서, 입금, 미수금)

### 추가 제안 기능 (고객 관리와 연동)
✅ **광고 계정 정보 관리** (고객별 광고 매체 계정)
✅ **보고서 관리** (고객별 보고서 작성 및 발송 이력)

### Phase 1 제외 항목 (Phase 2-3으로 이연)
- 프로젝트/캠페인 진행 관리
- 작업/일정 통합 캘린더
- 매체 예산 관리
- 시간 추적 & 수익성 분석
- 리드/영업 관리
- OCR 기능
- 자동 발주
- 크롤러 연동
- 광고 API 연동
- 외부 API 통합

## 💡 기대 효과

### 정량적 효과
