## 4. 테스트 계획

### 4.1 단위 테스트 (향후)
- **대상**: 서비스 레이어 함수
- **도구**: Jest
- **범위**: 비즈니스 로직 검증

### 4.2 통합 테스트 (향후)
- **대상**: API 엔드포인트
- **도구**: Jest + Supertest
- **범위**: API 요청/응답 검증

### 4.3 E2E 테스트 (향후)
- **대상**: 주요 사용자 플로우
- **도구**: Playwright
- **범위**: 
  - 고객 생성 → 주문 생성 → 정산
  - 견적서 생성 → 주문 변환
  - 작업 생성 → 시간 기록

### 4.4 수동 테스트 (MVP)
- **대상**: 모든 기능
- **방법**: 직접 테스트
- **체크리스트**: 각 기능별 테스트 시나리오 작성

## 5. 배포 계획

### 5.1 MVP 배포 환경
- **플랫폼**: Vercel (프론트엔드/백엔드)
- **데이터베이스**: Supabase 또는 Neon (PostgreSQL)
- **파일 저장**: 로컬 스토리지 또는 Vercel Blob

### 5.2 배포 전 체크리스트
- [ ] 환경 변수 설정
- [ ] 데이터베이스 마이그레이션 실행
- [ ] 빌드 테스트
- [ ] 프로덕션 빌드 확인
- [ ] 성능 테스트
- [ ] 보안 검토

### 5.3 배포 프로세스
1. **개발 환경 테스트**
   - 로컬에서 모든 기능 테스트
   - 데이터베이스 마이그레이션 확인

2. **스테이징 환경 배포** (선택사항)
   - 스테이징 서버에 배포
   - 통합 테스트 수행

3. **프로덕션 배포**
   - 프로덕션 환경 변수 설정
   - 데이터베이스 마이그레이션 실행

- 실제 개발 진행 상황에 따라 일정 조정 가능
- 우선순위에 따라 기능 범위 조정 가능

### 10.2 커뮤니케이션
- 정기적인 진행 상황 공유
- 이슈 발생 시 즉시 공유
- 피드백 수집 및 반영

### 10.3 문서 업데이트
- 개발 진행에 따라 문서 지속 업데이트
- 변경사항 문서화
