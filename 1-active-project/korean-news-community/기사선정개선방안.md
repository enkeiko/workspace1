# 기사 선정 개선 방안

## 📋 현재 상태

### 완료된 개선사항
1. ✅ **"적용 아이디어/팁" 제거** - 의미 없는 일반적인 조언 제거
2. ✅ **요약 품질 개선** - 더 자세한 요약 (400자까지 확장)
3. ✅ **링크 개선** - 제목에 링크 포함, 명확한 "전체 기사 읽기" 버튼
4. ✅ **필터링 로직 강화** - 임계값 상향 (3→4), 점수 차이 요구 (최소 2점)

### 개선된 출력 형식
- 제목에 직접 링크 포함
- 더 자세한 요약 (100-400자)
- 출처 및 점수 정보 표시
- 명확한 "전체 기사 읽기" 링크

## 🤖 AI 기반 기사 선정

### 현재 방식 (키워드 기반)
- **장점**: 빠르고 비용 없음
- **단점**: 
  - 키워드 매칭만으로는 맥락 파악 어려움
  - 예: "화물택배" 같은 기사가 AI 카테고리에 잘못 분류될 수 있음
  - 관련성 낮은 기사가 포함될 수 있음

### AI 기반 선정 (추가됨)
- **장점**:
  - 맥락 이해로 더 정확한 분류
  - 기사 품질 평가 가능
  - 관련성 높은 기사만 선정
- **단점**:
  - API 비용 발생 (무료 모델 사용 가능)
  - 처리 시간 증가 (기사당 0.5초)

## 🎯 권장 사용 방법

### 옵션 1: 키워드 기반 (현재, 기본)
```env
USE_AI_FILTER=false
```
- 빠르고 무료
- 대부분의 경우 충분히 정확
- 일부 엣지 케이스에서 오분류 가능

### 옵션 2: AI 기반 (권장, OpenRouter API 키 필요)
```env
USE_AI_FILTER=true
AI_FILTER_CONFIDENCE_THRESHOLD=70
OPENROUTER_API_KEY=sk-or-xxx
AI_MODEL=google/gemini-2.0-flash-exp:free
```
- 더 정확한 분류
- 품질 평가 포함
- 무료 모델 사용 가능 (Gemini 2.0 Flash)

### 옵션 3: 하이브리드 (최적)
- 키워드 기반으로 1차 필터링
- 애매한 기사만 AI로 재검토
- (아직 구현 안됨, 향후 추가 가능)

## 📊 기사 선정 기준 검토

### '돈되는 바이브코딩과 마케팅' 카테고리
**현재 키워드:**
- AI, ChatGPT, 자동화, 바이브코딩, SaaS, 노코드, 수익화, 부업 등

**검토 필요:**
- ✅ 대부분 적절함
- ⚠️ "화물택배 가격 비교" 같은 기사는 제외 필요
- ⚠️ 단순 기술 뉴스보다는 "수익화 가능성"이 있는 기사 우선

**개선 방안:**
- 수익/부업 관련 키워드 가중치 상향
- 단순 기술 소식보다 실용적 가이드 우선

### '자영업자 마케팅 및 네이버 플레이스' 카테고리
**현재 키워드:**
- 네이버 플레이스, 자영업, 소상공인, 리뷰 마케팅, 로컬 SEO 등

**검토 필요:**
- ✅ 대부분 적절함
- ⚠️ 금융 지원 정책 같은 기사는 관련성 낮을 수 있음
- ⚠️ 실제 마케팅/매출 증대 방법이 있는 기사 우선

**개선 방안:**
- 마케팅/매출 관련 키워드 가중치 상향
- 정책 소식보다 실전 팁 우선

## 🔧 설정 방법

### .env 파일에 추가
```env
# AI 기반 필터 사용 (선택사항)
USE_AI_FILTER=false  # true로 변경하면 AI 필터 사용
AI_FILTER_CONFIDENCE_THRESHOLD=70  # 신뢰도 임계값 (0-100)

# OpenRouter API (AI 필터 사용 시 필요)
OPENROUTER_API_KEY=sk-or-xxxxxxxxxxxxxxxx
AI_MODEL=google/gemini-2.0-flash-exp:free
```

## 📈 예상 효과

### 키워드 기반 (현재)
- 정확도: 약 75-80%
- 처리 속도: 매우 빠름
- 비용: 무료

### AI 기반 (추가됨)
- 정확도: 약 90-95%
- 처리 속도: 기사당 0.5초
- 비용: 무료 모델 사용 시 무료
