# 프로젝트 개요 및 요구사항 명세서

## 1. 프로젝트 목적 및 범위

### 1.1 프로젝트 목적
1인 마케팅 대행사 운영을 위한 통합 성과 및 정산 관리 시스템을 구축하여, 고객 관리부터 매출 정산까지의 전 과정을 효율적으로 관리하고 자동화하는 것을 목적으로 합니다.

### 1.2 프로젝트 범위
- **대상 사용자**: 1인 마케팅 대행사 운영자
- **주요 기능**: 고객 관리, 매장 관리, 구매/판매 관리, 정산 관리, 상담/보고서 관리
- **개발 범위**: MVP (Minimum Viable Product) 중심의 웹 애플리케이션
- **배포 환경**: 초기 로컬/클라우드 호스팅, 향후 확장 가능

### 1.3 프로젝트 목표
- 업무 프로세스 자동화 및 효율화
- 정확한 매출/비용 추적 및 정산 관리
- 고객별/매장별 성과 분석 및 리포트 생성
- 시간 추적 및 작업 관리 통합

## 2. 사용자 스토리 (User Stories)

### 2.1 고객 관리
- **US-001**: 고객 정보를 등록하고 관리할 수 있다
- **US-002**: 사업자등록증/명함을 OCR로 읽어 자동으로 정보를 입력할 수 있다
- **US-003**: 고객별 연결된 매장 목록을 확인할 수 있다
- **US-004**: 고객별 상담 히스토리를 조회할 수 있다
- **US-005**: 고객별 보고서 발송 이력을 확인할 수 있다

### 2.2 매장 관리
- **US-006**: 매장 기본 정보를 등록하고 관리할 수 있다
- **US-007**: 매장별 분석 정보를 기록하고 조회할 수 있다
- **US-008**: 매장별 작업 및 순위 히스토리를 추적할 수 있다
- **US-009**: 크롤러와 연동하여 매장 데이터를 자동 수집할 수 있다
- **US-010**: 매장별 연결된 플랫폼 계정을 관리할 수 있다

### 2.3 구매 관리
- **US-011**: 구매 제품 정보를 등록하고 관리할 수 있다
- **US-012**: 매입 단가, 트래픽, 블로그 작업 등 구매 항목을 기록할 수 있다
- **US-013**: 발주 조건을 설정하고 자동 발주할 수 있다
- **US-014**: 거래처 정보를 관리할 수 있다
- **US-015**: 발주 내역을 조회하고 관리할 수 있다

### 2.4 판매 관리
- **US-016**: 상품 정보를 등록하고 관리할 수 있다
- **US-017**: 주문을 등록하고 고객/매장과 연동할 수 있다
- **US-018**: 일/주/월/분기/연 단위로 주문을 조회할 수 있다
- **US-019**: 견적서를 생성하고 고객별 단가를 저장할 수 있다
- **US-020**: 세금계산서를 발송할 수 있다
- **US-021**: 계약서를 관리하고 핵심 내용을 추출할 수 있다

### 2.5 정산 관리
- **US-022**: 매출/정산 현황을 대시보드에서 한눈에 확인할 수 있다
- **US-023**: 기간별 매출과 비용을 조회할 수 있다
- **US-024**: 매장별/고객별 판매액을 집계할 수 있다
- **US-025**: 견적서와 세금계산서 발급 및 입금 여부를 확인할 수 있다
- **US-026**: 총매출, 총비용, 수익, 미수, 선금을 관리할 수 있다
- **US-027**: 카카오톡으로 견적서/세금계산서를 자동 발송할 수 있다

### 2.6 상담 및 보고서 관리
- **US-028**: 고객과의 상담 내용을 기록하고 조회할 수 있다
- **US-029**: 상담 채널별(카카오톡, 전화, 이메일 등)로 히스토리를 관리할 수 있다
- **US-030**: 보고서를 생성하고 고객에게 발송할 수 있다
- **US-031**: 보고서 템플릿을 관리하고 재사용할 수 있다
- **US-032**: 보고서 발송 이력을 추적할 수 있다

### 2.7 플랫폼 계정 관리
- **US-033**: 위탁받은 플랫폼 계정(네이버 플레이스, 구글 마이비즈니스 등)을 등록할 수 있다
- **US-034**: 계정별 위탁 기간을 관리할 수 있다
- **US-035**: 계정별 성과 지표를 확인할 수 있다

### 2.8 작업 및 시간 관리
- **US-036**: 작업을 등록하고 상태를 관리할 수 있다
- **US-037**: 작업별 소요 시간을 기록할 수 있다
- **US-038**: 프로젝트별/고객별 시간 집계를 확인할 수 있다
- **US-039**: 작업 우선순위를 설정할 수 있다

### 2.9 문서 관리
- **US-040**: 모든 문서를 중앙에서 관리할 수 있다
- **US-041**: 문서를 검색하고 필터링할 수 있다
- **US-042**: 문서를 다운로드할 수 있다

### 2.10 알림 및 리마인더
- **US-043**: 중요한 일정(마감일, 정산일 등)에 대한 알림을 받을 수 있다
- **US-044**: 미수금/미지급금 알림을 받을 수 있다
- **US-045**: 계약 만료 예정 알림을 받을 수 있다

## 5. MVP 범위 정의

### 5.1 MVP에 포함되는 기능

#### 핵심 기능 (Must Have)
1. **고객 관리**
   - 고객 기본 정보 CRUD
   - 고객별 매장 연결
   - 고객 목록 조회 및 다운로드

2. **매장 관리**
   - 매장 기본 정보 CRUD
   - 매장 분석 정보 기록
   - 매장 목록 조회 및 다운로드

3. **판매 관리**
   - 주문 등록 및 조회
   - 견적서 생성 (기본 템플릿)
   - 주문 상태 관리

4. **정산 관리**
   - 매출/정산 대시보드 (기본)
   - 기간별 매출/비용 조회
   - 미수금/선금 관리

5. **상담 관리**
   - 상담 히스토리 기록 및 조회
   - 상담 채널 관리

6. **작업 관리**
   - 작업 등록 및 상태 관리
   - 작업 시간 기록 (타임 트래킹)

7. **문서 관리**
   - 파일 업로드/다운로드
   - 문서 검색

8. **알림 시스템**
   - 기본 알림 기능

### 5.2 MVP에 제외되는 기능 (Phase 2 이후)

1. **OCR 기능**: 초기 수동 입력, 추후 자동화
2. **크롤러 연동**: 추후 구현
3. **전자계약 API**: 추후 구현
4. **은행계좌 API**: 추후 구현
5. **카카오톡 자동 발송**: 기본 기능만, 자동화는 추후
6. **보고서 자동 생성**: 수동 생성만, 자동화는 추후
7. **고객 포털**: 추후 구현
8. **다중 사용자**: 1인 운영 전제

### 5.3 MVP 성공 기준

- 고객 정보를 등록하고 관리할 수 있다
- 주문을 등록하고 정산할 수 있다
- 매출/정산 현황을 대시보드에서 확인할 수 있다
- 상담 히스토리를 기록하고 조회할 수 있다
- 작업 시간을 기록하고 집계할 수 있다
- 모든 데이터를 Excel로 다운로드할 수 있다

## 6. 제약사항 및 가정

### 6.1 제약사항
- 초기 개발은 1인 운영자 전제
- 외부 API 연동은 MVP 이후 단계적으로 구현
- 파일 저장은 로컬 스토리지 우선, 클라우드 저장소는 추후

### 6.2 가정
- 사용자는 기본적인 컴퓨터 사용 능력 보유
- 인터넷 연결 환경
- PostgreSQL 데이터베이스 사용 가능

## 7. 용어 정의

- **고객**: 마케팅 서비스를 받는 클라이언트
- **매장**: 고객이 운영하는 실제 매장 또는 온라인 매장
- **플랫폼 계정**: 네이버 플레이스, 구글 마이비즈니스 등 위탁받은 계정
- **견적서**: 고객에게 제공하는 서비스 견적
- **세금계산서**: 세금계산서 발행
- **미수금**: 아직 입금받지 못한 금액
- **선금**: 작업 전에 받은 선금
- **타임 트래킹**: 작업에 소요된 시간을 기록하는 기능
