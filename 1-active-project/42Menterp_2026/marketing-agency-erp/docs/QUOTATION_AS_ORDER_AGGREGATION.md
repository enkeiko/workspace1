# 견적서 = 여러 주문의 합

## 현재 구조 분석

### 견적서 구조
```
Quotation (견적서)
├── quotationNumber: "QT-202511-000001" (고유한 견적서번호)
├── customerId: 1 (고객)
├── storeId: 1 (매장)
├── totalAmount: 500,000원 (모든 항목의 합계)
├── items: QuotationItem[] (여러 항목)
│   ├── QuotationItem 1: 상품A, 수량 2, 단가 30,000원
│   ├── QuotationItem 2: 상품B, 수량 1, 단가 40,000원
│   └── QuotationItem 3: 커스텀 상품, 수량 1, 단가 30,000원
└── orders: Order[] (여러 주문을 가질 수 있음 - 1:N 관계)
```

### 현재 구현의 한계

1. **견적서 생성 시**:
   - 여러 항목을 직접 입력하거나
   - 하나의 주문에서 항목을 가져올 수 있음
   - 하지만 여러 주문을 선택해서 하나의 견적서로 묶는 기능은 없음

2. **주문 → 견적서 전환**:
   - 하나의 주문을 하나의 견적서로 전환
   - 여러 주문을 하나의 견적서로 묶는 기능은 없음

### 필요한 기능

1. **여러 주문 선택 → 견적서 생성**:
   - 주문 목록에서 여러 주문 선택
   - 선택한 주문들의 모든 항목을 합쳐서 하나의 견적서 생성
   - 선택한 주문들을 견적서에 연결

2. **견적서 = 여러 주문의 합**:
   - 견적서 총액 = 연결된 모든 주문의 총액 합계
   - 견적서 항목 = 연결된 모든 주문의 항목 합계

## 개선 방안

### 방안 1: 여러 주문 선택하여 견적서 생성
- 주문 목록 페이지에서 여러 주문 선택
- "견적서로 묶기" 버튼 클릭
- 선택한 주문들의 항목을 합쳐서 견적서 생성

### 방안 2: 견적서 생성 시 여러 주문 선택
- 견적서 생성 페이지에서 여러 주문 선택
- 선택한 주문들의 항목을 자동으로 견적서 항목으로 추가
- 주문들을 견적서에 연결

### 방안 3: 견적서 총액 자동 계산
- 견적서에 연결된 주문들의 총액을 합산
- 견적서 항목도 연결된 주문들의 항목을 합산
