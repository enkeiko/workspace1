last_updated: 2025-11-14

## A. 전략 프레임 (Relevance · Popularity · Trust)

### A-0. 원칙
**Fact**
- 네이버 검색·노출 알고리즘 가중치는 미공개 → **확인 불가**  
- 플레이스 정보 완성도(NAP·메뉴·사진·설명)는 검색 결과 구성 요소로 평가됨  
- 리뷰 신뢰도는 영수증/예약 인증 중심으로 판단됨  

**Interpretation**
- 플레이스 SEO는 순위 조작이 아니라  
  **사용자 이해도·신뢰도·전환률 개선을 위한 구조적 정비 작업**이다.

**Hypothesis**
- 클릭·체류·전환 흐름이 안정적일수록 노출 위치가 개선될 “가능성”이 있음  
  (정확 가중치는 미공개 → 보조 참고만 가능)

---

## A-1. 3대 축 구조

### 1) 적합도(Relevance)
- 업종/카테고리 정확성  
- 대표키워드–콘텐츠 일치  
- 메뉴/운영정보 정확성  

### 2) 인기도(Popularity)
- 클릭률(전화/길찾기/예약)  
- 리뷰 증가량  
- 소식 게시 빈도  

### 3) 신뢰도(Trust)
- 영수증 리뷰 비중  
- 사진 품질  
- 답글 SLA(24~48h)  

---

## B. 상호명 · 카테고리 전략

### B-1. 상호명
- 키워드 삽입 금지(정책 위반)  
- 구조: **브랜드명 + 지점명**  
- “본점/직영점” 표기 허용  
- CTR 실험은 콘텐츠 기반 간접 실험으로만 수행  

### B-2. 카테고리
**Fact**
- 네이버 공식 카테고리만 선택 가능  
- 기본 카테고리 1개가 핵심 검색 연관성을 결정  

**전략**
- 주력 업종을 기본 카테고리로 설정  
- 부가 업종은 부가정보 처리  
- 상권 내 동일 카테고리 매장 수는 확인 가능  
- 경쟁도 가중치는 **확인 불가 → 보조 참고만 가능**

### B-3. NAP 관리
- Collector → Analyzer로 정보 정합성 자동 검증  
- 외부 채널과 정보 100% 일치 유지  
- 변경 시 즉시 업데이트

## C. 키워드 전략 (정합 통합본)

### C-1. 구성 요소
- 대표키워드 ≤ 5개  
- 지역키워드  
- 속성키워드  
- 조합키워드  

### C-2. Fact 기반 규칙
- 필드에 키워드 삽입해 순위 상승한다는 **근거 없음**  
- 검색량 비교는 데이터랩만 공식 활용 가능  
- 비공식 경쟁지표는 **확인 불가**,  
  필요 시 별도 데이터로 **보조 참고만 가능**

---

## C-3. C-Sys (키워드 자동화 구조)

### C-Sys-0 | Meta 수집
- keyword_meta_taxonomy.yaml 기반  
  - 지역(Location)  
  - 대상(Target)  
  - 의도(Intent)  
  - 감성(Tone)  
  - 시점(Time/Season)

### C-Sys-1 | 매핑
- 리뷰·메뉴·소식·외부 채널에서 자연어 추출  
- 유사어 클러스터링  

### C-Sys-2 | 효율지수
```
Efficiency = log(검색량+1) / (경쟁도+1)
```
※ 경쟁도 = 상권 내 동일 업종/키워드 매장 수  
※ 가중치는 미공개 → **확인 불가**

### C-Sys-3 | Improver 연동
- tone/time/meta 기반 추천  
- 중복·스태핑 차단  
- 조합키워드 문장화

## D. 내부 콘텐츠 전략 (Intro · News · Visual)

### D-0. 원칙

**Fact**
- 소개문은 외부 문서(블로그/웹문서) 기반으로 노출됨  
- 과장/추정/스태핑 금지  
- 가격/운영정보는 NAP과 일치해야 함

**Interpretation**
- 내부 콘텐츠는 “신뢰–이해–전환”을 만드는 구조  

---

## D-1. 소개문 (Intro)
- 분량: 1,200~2,000자  
- 구조:  
  ① 브랜드 → ② 서비스 → ③ 이용가이드 → ④ 위치 → ⑤ 후기 → ⑥ 마무리  

**규칙**
- 대표키워드 1~2회  
- tone/time/meta 기반 문장 톤  
- 운영정보·가격 정확성 필수  
- 해시태그형 문장 금지  

---

## D-2. 소식 (News/Post)
- 주기: 월 2회  
- 제목(감성) + 본문(정보) + 이미지/영상  

**규칙**
- 감성/시즌 중심 주제  
- 문장형 키워드 삽입  
- 사실 기반(메뉴/가격) 유지  

---

## D-3. 시각콘텐츠 (Visual)
- 외관/내부/조리/완성/가격표/서비스 컷  
- 원본 고해상도 업로드  
- 파일명: `지역_메뉴_속성_번호.jpg`  
- 품질 기준: 중복 없음, 저해상 제거  

---

## D-Sys (자동화 흐름)
- Intro/News/Visual은 Analyzer → Improver로 tone/time/meta 최적화  
- 중복/오류 정합성 검증  
- BlogGuide 모듈로 연결  

---

## E-1. 외부 콘텐츠 (SNS/Short-form)

### 역할
- 내부는 정보·전환 중심  
- 외부는 감성·스토리 기반으로 **브랜드 검색량 증가** 역할

### 채널별 전략

**Instagram**
- 감성 기반 짧은 메시지  
- 이미지/스토리 기반  
- 해시태그 5~10개(감성/상황)

**TikTok**
- 3~7초 컷 편집  
- 메뉴·조리·반응 중심  
- CTA: “네이버에서 상호명 검색”

**YouTube**
- 30~60초 숏츠  
- 시그니처 메뉴·브랜드 스토리  

**주요 주의사항**
- 외부 노출량의 검색 순위 영향은 **확인 불가**,  
  다만 추세 분석은 보조 참고 가능  
- 과장·단정 문구 금지  

---

## E-2. 리뷰 전략

### Fact
- 영수증/예약 인증 리뷰 신뢰도 높음  
- 비정상 패턴(동일문장/동일IP)은 자동 미노출  
- 답글 SLA(24~48h)는 신뢰 신호로 보일 가능성 있음 (가중치는 미공개)

### 리뷰 유입
- 대가성 금지  
- 자연스러운 안내 멘트만 허용  
- QR 리뷰 안내 가능(업종 정책 확인)

### 리뷰 답글
- 사실 기반  
- 사과/설명 분리  
- 감정적 표현 금지  
- 재방문 유도는 중립적 톤  
- 분쟁성 리뷰는 정책에 따라 처리  

---

## E-Sys (External Sync Engine)

### E-Sys-1 입력
- SNS 텍스트/해시태그  
- 리뷰 표준 언어  
- 시즌/상황 정보  

### E-Sys-2 변환
- tone/time/meta 자동 추출  
- 감성/상황/대상 기반 패턴 분석  

### E-Sys-3 출력
- 소식 주제 추천  
- 소개문 보완 포인트  
- 리뷰 답글 방향성  
- 이미지 개선 포인트  

### E-Sys-4 KPI
- 브랜드 검색량(데이터랩)  
- 클릭률(전화/길찾기/예약)  
- 리뷰 증가량  
- 소식 CTR  

※ 외부 지표의 정확 가중치는 **확인 불가**,  
   보조 참고만 가능.
